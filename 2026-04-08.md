#### 简单说一说websocket的心跳机制：

-------

服务端和客户端约定一个暗号，服务器每隔10秒像客户端发送一个特殊的空消息(ping)。客户端收到ping后必须立刻自动回复一个消息(Pong)。服务器在代码里设定了一个“读取超时时间“，当服务端在指定时间内为收到(Pong)，则认定服务器断连，主动结束连接。

#### Websocket并发锁：

-----

在go中，Gorilla WebSocket 库有一个极其严格的底层规定：**一个连接只允许一个GoRoutine去写，一个GoRoutine去读**。假设三个人同时给一个帖子点赞，三个goroutine同时拿到连接对象，此时就会报错：`panic: concurrent write to websocket connection`

解决方案：

+ 互斥锁：谁先抢到锁谁发。
+ channel：开辟一个专属的gotoutine，将所有想要发送通知的goroutine放入其中，再由一个专门的”快递员“goroutine去队列中拿出消息，然后通过连接发出去。

#### 半关闭处理状态：

-------

也就是两边不能同时关闭连接，解决方案：

1. 客户端发送一个close控制帧
2. 服务器读到一个`CloseErroe`
3. 服务器此时不能直接断开连接，而是停止手头的业务去向客户端回发一个Close帧，表示我也同意断开
4. 双方完成握手，各自清理内存，断开TCP连接

#### 简单说一说函数闭包？

---

我对函数闭包的理解大概就是：一个函数+其引用的外部变量，打包在一起的整体：

+ 函数可以访问其定义的外部变量
+ 其引用的外部变量会和它一起存活，不会因为外层函数执行完毕就跟着销毁
+ 每次调用外部函数都会创建一个新的闭包

#### 接口你总懂点吧（唉啥都没准备就面试）

----

接口简单来说就是一组方法的声明，当某一类实现了该接口的所有方法，就称这种类型为该接口的一种实现

#### 空接口是什么

-----

没有任何方法被声明的接口称为**空接口**，它可以接受任意类型的参数：

```go
func main() {
    var any interface{}
    any = 10
    fmt.Println(any)

    any = "golang"
    fmt.Println(any)

    any = true
    fmt.Println(any)
}
/*
10
golang
true
*/
```

需要注意：任意类型的变量可以赋值给空接口，任意空接口不能赋值给其它任何变量

#### 简单说说defer怎么用？

---

defer主要做的是一个延迟执行，但也一定会执行，它是在`Return`时执行；它的执行符合现进后出；很经典的一个例子就是对于文件复制：

我们在打开一个文件之后可以立刻`defer f.close`。这样就算后续代码执行出了问题，文件也一定会执行关闭操作，可以保证资源不浪费。需要注意的是，defer内的参数在调用defer时就已经确定，后续对这个参数的修改都不会影响defer内已经固定的参数。但如果传给其的是一个地址，那么地址指向的变量进行修改的话，其值也会进行相应的修改。

#### goroutine 是多线程吗？

---

不完全是。goroutine 是用户态的轻量级协程，不是操作系统线程。它的栈初始只有 2KB，可以轻松创建百万个；而线程栈至少 1MB，创建几千个就很吃力。Go 运行时使用 GMP 模型，将多个 goroutine 调度到少量操作系统线程上执行，实现了高并发。从使用角度看，goroutine 像线程一样可以并发执行，但底层实现和性能特征完全不同。
